## 2026-06-05（项目搭建）

从零搭建基于 Spring Cloud Alibaba 的微服务社交平台。

### 一、后端架构搭建

#### 1. 微服务拆分（8 个服务）

| 服务 | 端口 | 职责 |
|------|------|------|
| social-gateway | 8080 | API 网关，路由转发、JWT 认证、跨域 |
| social-user | 8081 | 用户注册/登录、个人信息、头像、管理员 |
| social-post | 8082 | 动态发布/删除、点赞、评论、收藏 |
| social-social | 8083 | 关注/粉丝、好友申请、黑名单 |
| social-message | 8084 | 私信聊天（WebSocket）、系统通知、文件上传 |
| social-search | 8085 | 用户/动态/话题搜索 |
| social-statistics | 8086 | 平台数据统计 |
| social-ai | 8087 | AI 智能对话（智谱 GLM-4-Flash） |

#### 2. 公共模块（social-common）

- `social-common-core` — 响应封装（R 类）、全局异常处理、MyBatis-Plus 基类配置
- `social-common-redis` — Redis 操作工具类
- `social-common-security` — JWT 生成/校验工具、Gateway 认证过滤器

#### 3. 技术集成

- **Nacos**：8 个服务全部注册到 Nacos（localhost:8848）
- **Gateway**：统一入口，`lb://` 负载均衡路由，AuthFilter 校验 JWT 并传递 X-User-Id
- **RocketMQ**：发布动态时异步通知消息服务落库（Producer: social-post, Consumer: social-message）
- **Sentinel**：动态广场、搜索、AI 对话 3 个核心接口限流保护
- **Seata**：发布动态、接受好友申请 2 个场景分布式事务

#### 4. 数据库设计（13 张表）

`user`、`post`、`comment`、`like_record`、`collect`、`follow`、`friend`、`user_group`、`blacklist`、`message`、`ai_chat`、`search_history`、`statistics`

#### 5. 各服务核心实现

- **social-user**：BCrypt 密码加密、JWT 令牌生成、Redis 存储 token（24h 过期）
- **social-post**：动态 CRUD、点赞/评论/收藏、图片上传、@GlobalTransactional
- **social-social**：关注/取关、好友申请/接受/拒绝、@GlobalTransactional
- **social-message**：WebSocket 实时聊天、私信会话列表、在线状态查询
- **social-search**：关键词搜索用户/动态/话题、搜索历史
- **social-statistics**：每日新增用户/动态/活跃用户统计
- **social-ai**：接入智谱 GLM-4-Flash 免费模型、多轮对话上下文、会话管理

### 四、关键问题修复（搭建过程中）

- **MyBatis-Plus ID 策略冲突**：7 个服务的 `application.yml` 全局配置了 `id-type: ASSIGN_ID`（雪花算法），覆盖实体类的 `@TableId(type = IdType.AUTO)`，导致点赞/收藏对新动态不生效。修复：全部改为 `AUTO`。

## 2026-06-06（调试优化）

### 一、Bug 修复

#### 1. 管理页面服务器内部错误
- **问题**：管理页面访问时报 500 错误
- **原因**：用户角色为 `user` 而非 `admin`，权限不足
- **解决**：调整用户角色判断逻辑

#### 2. 在线状态接口 500 错误
- **问题**：`POST /api/message/online` 频繁报 500
- **原因**：群聊会话 ID（如 `group_1`）被错误传入在线状态查询接口，该接口期望 Long 类型
- **解决**：`Message.vue` 中 `checkConversationsOnline` 过滤掉群聊会话，只查询私聊在线状态

#### 3. 密码修改逻辑无效
- **问题**：旧密码填错也能修改成功
- **原因**：后端代码修改后未重新打包部署
- **解决**：重新 `mvn package` 并重启 social-user 服务

#### 4. 手机号校验过于严格
- **问题**：空手机号提交时报"手机号格式不正确"
- **原因**：`@Pattern` 正则不允许空字符串
- **解决**：正则改为 `^$|^1[3-9]\\d{9}$`，同时 service 层增加空字符串判断

#### 5. 铃铛红点逻辑问题
- **问题**：没有未读消息时铃铛仍显示红点
- **原因**：`v-if="notifCount"` 当 count 为 0 时仍为 truthy
- **解决**：改为 `v-if="notifCount > 0"` 严格数值比较；错误处理中重置 count 为 0

#### 6. 在线状态闪烁
- **问题**：在线状态图标"一闪一闪的"
- **原因**：每次轮询都无条件更新 `conv.online`，即使值未变也触发响应式更新
- **解决**：只在状态实际变化时才更新；在线检查频率从每 3 秒改为每 15 秒

#### 7. 通知 ID 仍为雪花 ID
- **问题**：数据库中多张表 ID 仍为超长雪花 ID
- **原因**：`AUTO_INCREMENT` 值未重置
- **解决**：清空相关表数据，所有表 `AUTO_INCREMENT` 重置为 100
