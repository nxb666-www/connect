# 社交平台 - 项目结构与功能清单

## 一、项目概述

基于 Spring Cloud Alibaba 微服务架构的社交平台，前后端分离，包含 8 个后端微服务和 1 个 Vue 3 前端应用。

## 四、微服务架构

### 服务列表与端口

| 服务 | 端口 | 职责 |
|------|------|------|
| social-gateway | 8080 | API 网关，路由转发、JWT 认证、跨域处理 |
| social-user | 8081 | 用户注册/登录、个人信息管理、头像上传、管理员功能 |
| social-post | 8082 | 动态发布/删除、点赞、评论、收藏 |
| social-social | 8083 | 关注/粉丝、好友申请、黑名单 |
| social-message | 8084 | 私信聊天（WebSocket）、系统通知、文件上传 |
| social-search | 8085 | 用户/动态/话题搜索 |
| social-statistics | 8086 | 平台数据统计 |
| social-ai | 8087 | AI 智能对话（接入智谱 GLM-4-Flash） |

## 五、数据库设计

数据库名：`social_platform`，字符集 `utf8mb4`，共 **13 张表**：

| 表名 | 说明 | 主要字段 |
|------|------|----------|
| `user` | 用户表 | username, password(BCrypt), nickname, avatar, email, phone, signature, level, points, role, status |
| `post` | 动态表 | user_id, content, images(JSON), topic, visibility(公开/好友/私密), like/comment/collect_count |
| `comment` | 评论表 | post_id, user_id, content, parent_id(支持嵌套回复), like_count |
| `like_record` | 点赞记录 | post_id, user_id（联合唯一） |
| `collect` | 收藏记录 | post_id, user_id（联合唯一） |
| `follow` | 关注关系 | user_id, follow_id（联合唯一） |
| `friend` | 好友关系 | user_id, friend_id, status(0=待确认, 1=已接受) |
| `user_group` | 好友分组 | user_id, name |
| `blacklist` | 黑名单 | user_id, black_id（联合唯一） |
| `message` | 私信/通知 | sender_id, receiver_id, content, type(1=私信, 2=系统), is_read |
| `ai_chat` | AI 对话记录 | user_id, session_id, role(user/assistant), content |
| `search_history` | 搜索历史 | user_id, keyword, type(user/post/topic) |
| `statistics` | 平台统计 | date, new_users, active_users, new_posts, total_likes, total_comments |

用户表使用自增ID（AUTO_INCREMENT），其他表使用雪花算法生成主键。
